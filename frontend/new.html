FIFA Player Stats with Zero-Knowledge Proof Integration

Project Overview

This project is a full-stack web application that allows users to fetch FIFA player stats and uses Zero-Knowledge Proof (ZKP) to verify specific player stats without revealing sensitive information. The goal is to demonstrate how privacy and cryptographic verification can work together to provide a more secure experience in gaming.

Key Features
  • Player Stats Fetching: Users can search for a player by name and fetch stats (OVA, POT, Position, etc.).
  • Zero-Knowledge Proof Integration: A **ZKP** mechanism is used to verify whether a player's Overall Rating (OVA) exceeds a threshold, without revealing the actual rating.
  • Error Handling: The app handles errors gracefully, such as invalid player names or API issues.
  • Real-Time Data Fetching: The frontend uses JavaScript to fetch data from the backend API dynamically.

System Architecture

The project has a client-server architecture consisting of two main components:

  • Frontend (Client): Built using HTML, CSS, and JavaScript. Allows users to input a player's name and interact with the stats.
  • Backend (Server): Node.js with Express.js serves the API routes to fetch player data from a CSV file. Handles player search and ZKP functionality.
Zero-Knowledge Proof (ZKP) Simulation

The Zero-Knowledge Proof (ZKP) in this project is implemented as a simple simulation. It verifies whether a player's Overall Rating (OVA) is above a specified threshold (80) without revealing the actual rating. The ZKP validation process ensures that:

  • The OVA is greater than the threshold.
  • The OVA value is never exposed to the user directly.

This simulation can be enhanced by integrating zkSNARKs or zkSTARKs to perform real-world cryptographic proof of player stats.

Future Enhancements
